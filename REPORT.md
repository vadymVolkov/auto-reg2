# Отчёт: Автоматизация регистрации AWS

## Цель
Автоматизировать процесс регистрации на AWS через форму Device Authorization Flow в headless режиме.

## Реализация

### Технологии
- **Python 3.14**
- **Selenium + undetected-chromedriver** — обход базовой детекции автоматизации
- **ExpressVPN SOCKS5 прокси** — маскировка IP-адреса (USA New York)
- **Имитация человеческого поведения** — задержки, скролл, движения мыши

### Реализованные функции

1. **Базовая автоматизация**
   - Открытие страницы с user_code
   - Принятие cookies
   - Ввод email
   - Нажатие Continue
   - Ввод имени
   - Нажатие Continue на второй странице
   - Сохранение скриншота и HTML третьей страницы

2. **Антидетекция**
   - undetected-chromedriver (обход navigator.webdriver)
   - Headless режим с флагами антидетекции
   - Человекоподобный ввод текста (посимвольно с задержками)
   - Случайные задержки между действиями

3. **Продвинутая антидетекция**
   - ExpressVPN SOCKS5 прокси (127.0.0.1:3114)
   - Имитация движений мыши
   - Имитация скролла страницы
   - Имитация чтения страницы (5-15 секунд)
   - Увеличенные задержки (30-60 секунд на весь процесс)
   - Клики на поля ввода (фокус)
   - Паузы при вводе текста

## Анализ блокировки

### AWS TES (Threat Exchange Service)

AWS использует многоуровневую систему защиты от ботов:

1. **Браузерные отпечатки** ✓ Обходим
   - navigator.webdriver
   - Chrome runtime
   - Plugins, languages

2. **Поведенческий анализ** ✓ Обходим частично
   - Скорость ввода
   - Движения мыши
   - Скролл

3. **IP-репутация** ✓ Обходим
   - ExpressVPN прокси

4. **Серверная аналитика TES** ❌ НЕ обходим
   - Timing attacks
   - Порядок загрузки ресурсов
   - Метаданные сессии (скрытое поле metadata1)
   - История браузера и cookies
   - Headless детекция на уровне WebGL/Canvas
   - Паттерны сетевых запросов

### Скрытое поле metadata1

В форме присутствует скрытое поле с зашифрованными данными:
```html
<input name="metadata1" type="hidden" value="ECdITeCs:...">
```

Это поле содержит fingerprint сессии, который AWS анализирует на стороне сервера.

## Выводы

### Почему блокируется

AWS TES определяет автоматизацию по признакам, которые невозможно обойти на клиентской стороне:

1. **Headless режим** — даже с undetected-chromedriver AWS может определять headless через WebGL/Canvas fingerprinting
2. **Новый профиль браузера** — отсутствие истории, cookies, расширений
3. **Серверные метрики** — порядок и timing загрузки ресурсов отличается от реального браузера
4. **Metadata fingerprint** — скрытое поле содержит данные, которые AWS проверяет на сервере

### Что можно попробовать (требует дополнительных ресурсов)

1. **Residential прокси** — использовать прокси с IP реальных пользователей (платные сервисы)
2. **Реальный браузер с GUI** — запуск на машине с графическим интерфейсом
3. **Профиль с историей** — использовать существующий профиль Chrome с историей посещений
4. **Ручная регистрация** — самый надёжный способ

## Заключение

Техническая реализация автоматизации выполнена корректно. Скрипт успешно:
- Открывает страницу
- Заполняет форму
- Проходит через две страницы
- Сохраняет результаты

Однако AWS TES блокирует запрос на уровне API (`/api/send-otp`), используя серверную аналитику, которую невозможно обойти стандартными методами автоматизации браузера.

Для успешной автоматизации регистрации на AWS потребуются более сложные методы (residential прокси, реальный браузер с GUI, профиль с историей) или ручная регистрация.
